## 人工智能辅助形式化

人工智能工具在整个形式化过程中发挥了重要作用。本文主要使用的是两个系统，Codex [OpenAICodex2025] 和 ChatGPT 5.5 Pro [OpenAIChatGPTPro2026]，计算成本包括一个 ChatGPT Pro 5x 订阅（每个订阅每月 100 美元），并消耗了约 50% 的每周配额。

这些工具在多个层面上协助了形式化过程：

**自然语言对话。** ChatGPT 用来为形式化提供数学依据，包括以下方面：

- **检查数学证明的正确性。** 在将非形式化证明转写为 Lean 代码之前，ChatGPT 可用于检查原始数学推导是否存在逻辑跳步、条件遗漏或符号使用不一致等问题。例如，在处理理想分解、类群阶数和单位群结构时，ChatGPT 可以帮助确认每一步数学推理是否能够作为后续形式化的依据。
- **引理发现。** 在形式化过程中，从 Mathlib 庞大的 API 中搜索合适的引理是一个众所周知的瓶颈。ChatGPT 通常能够从所需结果的自然语言描述中识别出相关的引理，从而显著缩短搜索时间。
- **提出形式化方案。** 在面对一个数学结论时，ChatGPT 可以帮助判断应当采用哪种 Lean 表达方式，例如使用理想、商群、类群、环同态，还是转化为整数上的等式证明。这类方案设计有助于在正式写代码之前确定较合理的形式化路线。

**代码生成与修复。** Codex 直接参与代码产生过程，包括以下方面：

- **生成初始代码脚手架。** Codex 可以根据已经整理好的数学证明结构，生成 Lean 文件的初始框架，包括模块划分、定理声明、变量声明和必要的 import。对于较长的证明，Codex 通常先生成带有待证明目标的代码结构，再逐步补全具体证明。
- **生成重复性证明代码。** 在一些结构相似的分类讨论中，例如单位群情形划分、同余类讨论、模条件下的若干分支证明等，Codex 可以快速生成多个相似分支的证明代码。虽然这些代码仍需要人工检查和局部修改，但整体上明显提高了开发效率。
- **根据编译错误修复证明。** 当 Lean 报告类型不匹配、定理名称错误、参数顺序不一致或 tactic 失败时，Codex 可以根据错误信息修改代码。尤其是在 Mathlib 更新导致 API 名称或定理签名变化时，Codex 能够根据新的错误信息尝试完成迁移和修复。

本章补充说明了本文 Lean 4 形式化工作的整体结果、项目结构和验证方式。人工智能工具提高了证明开发效率，但最终正确性仍以 Lean 内核检查和人工复核为准。
